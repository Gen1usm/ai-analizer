# Полный отчет по проекту AnalyzAI

## 1. Общая информация о проекте

**Название проекта:** AnalyzAI  
**Формат продукта:** веб-платформа для анализа научных отчетов  
**Тип решения:** AI-сервис для предварительной экспертной оценки и сопровождения научных материалов

AnalyzAI представляет собой веб-сайт, который помогает пользователю загружать или вставлять текст научного задания и научного отчета, после чего система формирует структурированный результат анализа. Платформа ориентирована на автоматизацию первичной экспертной проверки материалов перед подачей на конкурс, грант, внутреннюю научную экспертизу или защиту проекта.

Основная идея проекта заключается в том, чтобы сократить время на ручную проверку научных текстов, повысить качество подготовки отчетов и дать пользователю понятную обратную связь по сильным и слабым сторонам документа.

## 2. Назначение сайта

Сайт предназначен для:

- анализа соответствия научного отчета требованиям задания;
- оценки документа по ключевым критериям;
- выявления проблем в структуре и содержании;
- определения сильных и слабых сторон проекта;
- формирования рекомендаций по улучшению текста;
- проведения базового анализа рыночной применимости;
- отображения вероятности успешного прохождения экспертной оценки;
- предоставления AI-ассистента для дополнительных вопросов по проекту.

Таким образом, сайт работает как цифровой помощник исследователя, студента, преподавателя, научного сотрудника или проектной команды.

## 3. Основной пользовательский сценарий

Пользователь заходит на сайт и выполняет следующие шаги:

1. Вводит текст задания вручную или загружает файл.
2. Вводит текст отчета вручную или загружает файл.
3. Нажимает кнопку анализа.
4. Получает страницу результата с основными метриками и пояснениями.
5. При необходимости задает дополнительные вопросы встроенному AI-ассистенту.

Такой сценарий делает продукт понятным даже для неподготовленного пользователя и не требует отдельного обучения.

## 4. Реализованный функционал

На текущий момент в проекте реализованы следующие функции:

### 4.1. Загрузка и обработка данных

Система принимает данные в двух форматах:

- вставка текста вручную;
- загрузка файлов.

Поддерживаются следующие типы файлов:

- `.txt`
- `.docx`
- `.pdf`
- `.pptx`
- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

Для изображений используется OCR-распознавание текста через `pytesseract`.

### 4.2. Аналитическая страница результата

После обработки пользователь получает итоговую страницу, на которой отображаются:

- итоговый балл;
- соответствие шаблону;
- вероятность успеха;
- оценки по критериям;
- диаграммный блок по критериям;
- сильные и слабые стороны;
- рыночный анализ;
- языковые замечания;
- блок антиплагиата;
- блок проверки на ИИ.

### 4.3. Критерии оценки

В проекте заложены следующие критерии с весами:

- стратегическая релевантность — 20%;
- цель и задачи — 10%;
- научная новизна — 15%;
- практическая применимость — 20%;
- ожидаемые результаты — 15%;
- социально-экономический эффект — 10%;
- реализуемость — 10%.

Итоговый балл вычисляется как взвешенная сумма баллов по каждому критерию.

### 4.4. AI-ассистент

На сайте присутствует отдельный AI-ассистент, который может:

- отвечать на вопросы по отчету;
- помогать с формулировками;
- пояснять критерии оценки;
- подсказывать, как улучшить текст;
- учитывать переданный пользователем контекст задания и отчета.

### 4.5. Интерфейс и доступность

На сайте реализованы:

- светлая и темная тема;
- переключение языков `RU / KZ / EN`;
- адаптивная верстка;
- отдельная страница результата;
- удобная форма ввода и просмотра аналитики.

## 5. Технологический стек

Проект построен на следующих технологиях:

- **Backend:** Django 6.0.4
- **Frontend:** HTML, CSS, JavaScript
- **AI-интеграция:** OpenAI API
- **Работа с документами:** `python-docx`, `pypdf`, `python-pptx`
- **Обработка изображений:** `Pillow`
- **OCR:** `pytesseract`
- **Конфигурация окружения:** `python-dotenv`

Такой стек выбран как практичный для MVP: он быстро разворачивается, понятен в поддержке и позволяет легко расширять функциональность.

## 6. Архитектура решения

Логика сайта организована по модульному принципу:

- `views.py` обрабатывает пользовательские запросы и маршруты;
- `forms.py` отвечает за форму ввода;
- `file_parser.py` извлекает текст из разных типов файлов;
- `assistant_service.py` обеспечивает работу AI-ассистента;
- `ai_service.py` отвечает за аналитический блок;
- `scoring/__init__.py` хранит веса критериев и логику расчета итогового балла;
- HTML-шаблоны формируют страницы ввода и результата;
- статические файлы CSS/JS отвечают за внешний вид и клиентскую логику.

Такая структура удобна для последующей доработки, масштабирования и командной разработки.

К сильным сторонам AnalyzAI можно отнести:

- поддержку нескольких форматов файлов;
- наличие мультиязычного интерфейса;

- наличие отдельного AI-ассистента;
- сочетание экспертной оценки, текстового анализа и визуализации результатов;
- возможность дальнейшего масштабирования под образовательные и научные учреждения.

## 8. Ограничения текущей версии

Важно честно отметить, что текущая версия является **MVP** и имеет ряд ограничений:

- основной аналитический сервис `ai_service.py` сейчас реализован как упрощенная версия и не использует весь описанный в интерфейсе потенциал полного AI-анализа;
- часть заявленных аналитических блоков зависит от структуры данных, которые должна возвращать внешняя AI-логика;
- точность антиплагиата и определения ИИ зависит от реализации модуля текстового анализа;
- рыночный анализ пока имеет характер предварительной экспертной оценки, а не полноценного бизнес-исследования;
- проекту требуется дополнительное тестирование на реальных пользовательских данных.

Это не является недостатком идеи, а отражает нормальное состояние хакатонного прототипа, который уже демонстрирует ценность, но нуждается в следующем этапе развития.
